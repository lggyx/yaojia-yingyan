# 药价鹰眼 Demo 脚本

## 1. 开场

药价鹰眼面向医保价格治理场景，把价格异常发现、证据解释、对抗校验和处置闭环串成一条可演示链路。演示重点不是“AI 自动拍脑袋判案”，而是规则可复现、证据可追溯、结论可反驳、处置可闭环。

## 2. 启动

```powershell
bun install
bun run seed
bun run dev
```

打开 `http://localhost:5173`。

## 3. 监管大屏

先展示 KPI：监测记录数、异常候选、高风险数量、闭环率。随后切到价格明细表，说明系统自动执行 `/api/detect`，并把 S1-S4/S6 等剧本记录纳入异常候选。

## 4. 价格趋势

点击一条价格记录，右侧趋势图展示同医保编码、同地区的历史价格。用趋势解释“当前价不是孤立数字”，而是和历史、集采价、区域均价共同形成异常证据。

## 5. 异常详情

点击带风险标识的记录，打开异常详情。先看命中维度，再点击“推理取证”。页面展示 Agent 调用的工具链，例如中标价、历史价、区域均价、替代药。

## 6. 对抗校验

点击“红队校验”。说明系统会主动尝试反驳异常：规格剂型是否不可比、是否政策调价、是否特殊采购、是否单位录入错误。

推荐演示两类结果：

- S1：多数反驳失败，置信度高，进入确认异常。
- S6：规格/政策解释成立，置信度下降，进入误报排除。

## 7. 闭环处置

在看板区展示工单列：待处置、处置中、已整改、已闭环。创建或已有工单可以推进状态，并点击“复核”查看整改后价格是否回到阈值内。复核通过后推进到闭环。

## 8. 收尾

总结系统边界：规则引擎负责稳定检测，LLM 负责解释表达和辅助研判；最终状态变化、证据和工单事件都落库，适合监管复盘。
