## 0. 一句话定位

> **药价鹰眼**是一个面向医保部门的自主智能体：它实时**监测**药品与耗材价格流，自动**多维比价**发现异常，用大模型**解释"为什么异常 + 风险分级"**，一键**生成核查工单与处置建议**，并**跟踪至闭环**——一个 Demo 打通赛题"数据标化治理 / 动态监测 / 异常预警与闭环处置 / 集采价格落地跟踪"四个重点方向。

## 1. 赛道分析与选型决策（为什么是这个方向、这个方案）

### 1.1 四大方向横向对比

| 方向 | 报名密度 | Demo 观赏性 | 落地难度(8h) | 数据依赖 | 结论 |
|---|---|---|---|---|---|
| 基金监管 | **高**（最热门、队伍最多） | 高 | 中 | 高 | 竞争激烈，红海 |
| **医药价格治理** | **最低** | 高 | 中 | 中（Mock 可撑） | **首选** |
| 医保经办服务 | 中 | 中 | 低 | 中 | 题目易发散、难做深 |
| 数据赋能 | 中 | 中 | 高 | 高 | 开放式、易"做不深" |

**关键规则**：每个方向**各决出 6 支队伍**，确保各赛道都有奖、不会被某一方向"吃掉"。
**推论**：在"保 6 席"的规则下，**报名人数最少的方向 = 单位努力进决赛的概率最高**。医药价格治理正是报名最少的方向 → **性价比最高的赛道**。

### 1.2 为什么在医药价格治理里选"异常预警 + 闭环处置"

赛题原文重点方向（务必复用其原词以贴合评分）：

> 医药价格数据标化治理、药品及医用耗材价格智能采集与动态监测、价格异常预警与闭环处置、集采价格落地跟踪。

候选方案对比：

| 方案 | 覆盖重点方向 | "智能体味" | Demo 戏剧性 | 风险 |
|---|---|---|---|---|
| **A. 异常预警 + 闭环处置** | **监测 + 预警 + 闭环 + 集采落地 + 标化内嵌（4/4）** | **强** | **强** | 低 |
| B. 数据标化治理 | 标化（1/4，但是地基） | 中 | 中 | 低-中 |
| C. 集采落地跟踪 | 集采落地（1/4） | 中-高 | 中-高 | 低 |

**选 A 的三条硬理由：**

1. **最像"智能体"**：完整呈现 `感知 → 推理 → 决策 → 行动 → 反馈` 闭环 + 工具调用，正中"智能体大赛"主题。
2. **覆盖面最广**：A 天然把 B（标准化作为内嵌工具）和 C（集采落地作为一种异常类型）纳入其中，一个 Demo 覆盖赛题 4 个重点方向（标化治理作为内嵌工具体现）→ 评委对着评分表逐项打勾。
3. **用词与赛题完全一致**："异常预警""闭环处置""集采落地"是赛题原词 → 评分天然加分。

## 2. 产品定位与核心叙事

**用户**：医保局价格招采监管人员。
**痛点**：药价虚高、地区价差悬殊、集采中选药不落地、数据多源异构形成"数据孤岛"，人工筛查海量价格数据效率低、发现晚、处置链条断。
**价值**：让监管从"人工抽查、事后发现"升级为"智能体全量监测、实时预警、自动取证、一键处置、闭环跟踪"，守护医保基金、压实集采落地、挤出药价水分。

## 3. 智能体能力闭环（评委最看重的"Agent 味"）

```
 感知 Perceive ─→ 推理 Reason ─→ 决策 Decide ─→ 行动 Act ─→ 反馈闭环 Feedback
   监测价格流      自主调用工具      偏离度→风险    生成处置工单    复核是否修正
                   取证再推理        分级(高/中/低)
```

### 3.1 这是"工具调用型 Agent"（ReAct 式），不是聊天框

智能体面对一条疑似异常记录时，**自主决定调用以下工具取证，再据证据推理**：

| 工具 | 作用 | 实现（8h） |
|---|---|---|
| `standardize_code` 药品/耗材编码标准化 | 把多源异构描述映射到统一医保编码、识别"四同药品" | 规则 + 字典匹配，体现"标化治理" |
| `get_tender_price` 查集采中标价 | 取该品种集采中选价/约定采购量 | Mock 数据查表 |
| `get_history_price` 查历史价格 | 取该品种历史价格序列 | Mock 数据查表 |
| `get_regional_avg` 查跨区域均价 | 取同通用名同厂家周边省份均价 | Mock 数据聚合 |
| `get_alternatives` 查同类替代药 | 取同类可替代品种价格 | Mock 数据查表 |
| `create_work_order` 生成处置工单 | 产出结构化核查工单/处置建议 | 后端写入内存状态机 |

> 设计要点：即使部分工具底层是 Mock 查表，**"自主选择工具→取证→推理→行动"的编排过程**才是评委打分点。推理轨迹要在前端可视化展示。

## 4. 系统架构（Node 全家桶，Bun 管理）

### 4.1 技术栈

| 层 | 选型 | 理由 |
|---|---|---|
| 包管理 / 运行时 | **Bun + Bun workspaces** | 单仓多包，安装/启动极快 |
| 后端 | **Hono**（Bun 原生运行） | 轻量、稳定、零配置、DX 好 |
| 前端 | **React + Vite + TypeScript + Tailwind CSS** | 快速搭监测大屏，团队顺手 |
| 图表/大屏 | **ECharts**（apache-echarts / echarts-for-react） | 价格趋势、偏离度、看板可视化观感强 |
| LLM 客户端 | `fetch` 调 **OpenAI 兼容接口**（内网 DeepSeek / 通义千问 / 专用医疗大模型） | 可信空间内网部署；带**模板兜底** |
| 数据/状态 | **JSON Mock 文件 + 内存 Map** | 8h 无需数据库 |

### 4.2 单仓结构（Bun workspaces）

```
yaojia-yingyan/                # 药价鹰眼 monorepo（bun）
├─ package.json                # workspaces: ["apps/*","packages/*"]
├─ apps/
│  ├─ api/                     # Hono 后端
│  │  ├─ src/index.ts          # 路由入口
│  │  ├─ src/rules/            # 多维比价规则引擎
│  │  ├─ src/agent/            # Agent 编排(ReAct) + 工具集 + LLM 客户端
│  │  └─ src/data/             # mock 数据加载
│  └─ web/                     # React + Vite 监测大屏
│     └─ src/                  # 监测表 / 推理轨迹 / 闭环看板
└─ packages/
   └─ shared/                  # 共享 TS 类型 + mock 数据生成器
      ├─ types.ts              # PriceRecord / Anomaly / WorkOrder ...
      └─ mock/                 # 预置数据 + 植入异常
```

### 4.3 后端 API 端点

| 方法 | 路径 | 作用 |
|---|---|---|
| GET | `/api/prices` | 返回价格数据流（监测表） |
| POST | `/api/detect` | 跑规则引擎，返回异常列表（含偏离度、风险分级） |
| POST | `/api/agent/investigate/:id` | 对某异常启动智能体：返回**推理轨迹**（工具调用序列+证据）+ LLM 解释 + 处置建议 |
| POST | `/api/disposition/:id` | 生成/推进处置工单（状态机流转） |
| GET | `/api/board` | 闭环看板数据（待处置/处置中/已闭环） |

### 4.4 关键设计：规则保准确，LLM 保表达

- **异常检测 = 确定性规则**：植入的异常 100% 被检出 → Demo 绝不翻车。
- **LLM 只基于结构化证据生成文字**（解释 + 工单），接地气、低幻觉。
- **LLM 兜底**：内网模型连不上时，用模板生成解释文本，Demo 照常跑（`MOCK_LLM=1` 开关）。

## 5. 多维比价规则引擎

| 维度 | 比价基准 | 异常判定（示例阈值，可调） | 对应赛题重点 |
|---|---|---|---|
| 集采落地价 | 集采中标价 | 实际采购价 / 中标价 > 1.5 | 集采落地 + 预警 |
| 历史价 | 该品种历史均价/中位数 | 环比涨幅 > 30% | 动态监测 |
| 跨区域 | 同通用名同厂家周边省份均价 | 偏离 > 50% | 动态监测 + 地区价差 |
| 同类替代 | 同类可替代药价格带 | 显著高于替代药 + 替代药挤占中选药 | 集采落地 |
| 采购量落地 | 约定采购量 | 完成率 < 60% | 集采落地跟踪 |
| 标准化冲突 | 统一医保编码 | 同"四同药品"不同名/不同价 | 数据标化治理 |

**风险分级**：综合各维度偏离度加权 → 评分 → **高 / 中 / 低**三级（高=红、中=橙、低=黄）。

## 6. Mock 数据与植入异常（一表覆盖赛题重点）

预置约 50–100 条价格记录（药品 + 耗材），字段：`医保编码 / 通用名 / 商品名 / 厂家 / 规格 / 剂型 / 采购价 / 挂网价 / 集采中标价 / 约定采购量 / 实际采购量 / 医院 / 区域 / 日期`。其中**植入 5 个鲜活异常**：

| # | 植入异常 | 触发维度 | 映射赛题重点 |
|---|---|---|---|
| 1 | 某药某院采购价 **高于集采中标价 320%** | 集采落地价 | 集采落地 + 异常预警 |
| 2 | 同通用名同厂家，A 省比周边 3 省均价 **高 2.1 倍** | 跨区域 | 动态监测 + 地区价差 |
| 3 | 某药历史平稳后 **突然跳涨** | 历史价 | 动态监测 |
| 4 | 集采中选药采购量 **仅达约定量 30%**，被高价替代药挤占 | 采购量 + 替代 | 集采落地跟踪 |
| 5 | 同一药品 **不同商品名/规格未标化**、价差悬殊 | 标准化冲突 | 数据标化治理（内嵌工具体现）|

## 8. Demo 脚本（3–5 分钟）

1. **痛点开场**：药价虚高、地区价差、集采不落地、数据孤岛。
2. **监测大屏**：价格数据流入，系统自动标红异常。
3. **点开异常**（"采购价高于中标价 320%"）→ 展示**智能体推理轨迹**：自主调用比价工具取证 → LLM 生成「风险等级：高；依据：同通用名同厂家周边 3 省均价×0.3；疑似未按中标价采购」。
4. **一键生成核查工单** → 推入闭环看板。
5. **模拟处置** → 价格修正 → 智能体**复核** → **闭环关闭**。
6. **收尾**：覆盖赛题 4 重点方向 + 真·Agent 闭环 + 接真实数据即可上线。
