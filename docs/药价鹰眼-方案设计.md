## 0. 一句话定位

> **药价鹰眼**是一个面向医保部门的自主智能体：它实时**监测**药品与耗材价格流，自动**多维比价**发现异常，经**对抗校验（红队自我反驳）算出置信度、剔除误报**，用大模型**解释"为什么异常 + 风险分级"**，一键**生成核查工单与处置建议**，并**跟踪至闭环**——一个 Demo 打通赛题"数据标化治理 / 动态监测 / 异常预警与闭环处置 / 集采价格落地跟踪"四个重点方向。

## 1. 赛道分析与选型决策（为什么是这个方向、这个方案）

### 1.1 四大方向横向对比

| 方向 | 报名密度 | Demo 观赏性 | 落地难度(8h) | 数据依赖 | 结论 |
|---|---|---|---|---|---|
| 基金监管 | **高**（最热门、队伍最多） | 高 | 中 | 高 | 竞争激烈，红海 |
| **医药价格治理** | **最低** | 高 | 中 | 中（Mock 可撑） | **首选** |
| 医保经办服务 | 中 | 中 | 低 | 中 | 题目易发散、难做深 |
| 数据赋能 | 中 | 中 | 高 | 高 | 开放式、易"做不深" |

**关键规则**：每个方向**各决出 6 支队伍**，确保各赛道都有奖、不会被某一方向"吃掉"。
**推论**：在"保 6 席"的规则下，**报名人数最少的方向 = 单位努力进决赛的概率最高**。医药价格治理正是报名最少的方向 → **性价比最高的赛道**。

### 1.2 为什么在医药价格治理里选"异常预警 + 闭环处置"

赛题原文重点方向（务必复用其原词以贴合评分）：

> 医药价格数据标化治理、药品及医用耗材价格智能采集与动态监测、价格异常预警与闭环处置、集采价格落地跟踪。

候选方案对比：

| 方案 | 覆盖重点方向 | "智能体味" | Demo 戏剧性 | 风险 |
|---|---|---|---|---|
| **A. 异常预警 + 闭环处置** | **监测 + 预警 + 闭环 + 集采落地 + 标化内嵌（4/4）** | **强** | **强** | 低 |
| B. 数据标化治理 | 标化（1/4，但是地基） | 中 | 中 | 低-中 |
| C. 集采落地跟踪 | 集采落地（1/4） | 中-高 | 中-高 | 低 |

**选 A 的三条硬理由：**

1. **最像"智能体"**：完整呈现 `感知 → 推理 → 决策 → 行动 → 反馈` 闭环 + 工具调用，正中"智能体大赛"主题。
2. **覆盖面最广**：A 天然把 B（标准化作为内嵌工具）和 C（集采落地作为一种异常类型）纳入其中，一个 Demo 覆盖赛题 4 个重点方向（标化治理作为内嵌工具体现）→ 评委对着评分表逐项打勾。
3. **用词与赛题完全一致**："异常预警""闭环处置""集采落地"是赛题原词 → 评分天然加分。

## 2. 产品定位与核心叙事

**用户**：医保局价格招采监管人员。
**痛点**：药价虚高、地区价差悬殊、集采中选药不落地、数据多源异构形成"数据孤岛"，人工筛查海量价格数据效率低、发现晚、处置链条断。
**价值**：让监管从"人工抽查、事后发现"升级为"智能体全量监测、实时预警、自动取证、一键处置、闭环跟踪"，守护医保基金、压实集采落地、挤出药价水分。

## 3. 智能体能力闭环（评委最看重的"Agent 味"）

```
感知 Perceive → 推理 Reason → 对抗校验 Challenge → 决策 Decide → 行动 Act → 反馈 Feedback
  监测价格流    调用工具取证    红队自我反驳·算置信度   风险分级    生成处置工单   复核是否修正
                                (反脆弱·降误判)      (含置信度)
```

### 3.1 这是"工具调用型 Agent"（ReAct 式），不是聊天框

智能体面对一条疑似异常记录时，**自主决定调用以下工具取证，再据证据推理**：

| 工具 | 作用 | 实现（8h） |
|---|---|---|
| `standardize_code` 药品/耗材编码标准化 | 把多源异构描述映射到统一医保编码、识别"四同药品" | 规则 + 字典匹配，体现"标化治理" |
| `get_tender_price` 查集采中标价 | 取该品种集采中选价/约定采购量 | SQLite 查表 |
| `get_history_price` 查历史价格 | 取该品种历史价格序列 | SQLite 查表 |
| `get_regional_avg` 查跨区域均价 | 取同通用名同厂家周边省份均价 | SQLite 聚合 |
| `get_alternatives` 查同类替代药 | 取同类可替代品种价格 | SQLite 查表 |
| `challenge_anomaly` 对抗校验 | 红队角色尝试反驳异常、算置信度（见 §3.2） | LLM(红队 prompt) + 重查证据工具 |
| `create_work_order` 生成处置工单 | 产出结构化核查工单/处置建议 | 写入 SQLite 状态机 |

> 设计要点：即使部分工具底层是查表，**"自主选择工具→取证→推理→对抗→行动"的编排过程**才是评委打分点。推理与对抗轨迹要在前端可视化展示。

### 3.2 对抗校验环节（提升置信度、降低误判）—— 本方案差异化亮点

医保价格监管最怕"误判"：很多价格差异其实有正当理由（规格/剂型不同不可比、政策性调价、特殊议价采购、新药上市合理溢价、数据口径/单位错误）。因此在"推理"与"决策"之间插入一个 **对抗校验（红队自我反驳）** 环节，把"简单阈值告警"升级为"经得起反驳的高置信预警"：

- **质疑者角色（Challenger / 红队）**：独立的 LLM 角色，专门尝试 **反驳** 该异常，逐条检验"是否存在让这个价格合理的解释"：
  1. 规格/剂型不同导致不可比？（重调 `standardize_code` 复核"四同"口径）
  2. 处于政策性调价 / 集采切换过渡期？
  3. 特殊采购情形（急救药、短缺药、谈判药、单一来源）？
  4. 数据录入口径 / 单位（如 盒 vs 支 vs 毫克）错误？
  5. 仍在同类替代药的正常价格带内？
- **置信度计算**：每条反驳被"挡回"则置信度上升、被"驳倒"则下降 → 输出 **置信度 0–100% + 反驳点清单 + 调整后风险等级**。
- **结果分流**：
  - 置信度高（≥80%）→ 升级 **高置信预警**，进入处置闭环。
  - 置信度中（50–80%）→ 标记 **待人工复核**。
  - 被驳倒（<50%）→ **自动消解（误报）**，记录消解理由，不打扰监管人员。

> 评委叙事：**"智能体先自我质疑、对抗校验，确认不是规格差异/政策调价后才升级预警"** —— 显著降低误报、让监管结论可信，是区别于普通告警系统的核心亮点。Demo 专门演示一条「看似异常、被对抗校验驳回」的诱饵案例（见 §6 第 6 条），凸显严谨与稳定。

### 4.5 关键设计：规则保准确，LLM 保表达

- **异常检测 = 确定性规则**：植入的异常 100% 被检出 → Demo 绝不翻车。
- **LLM 只基于结构化证据生成文字**（解释 + 工单），接地气、低幻觉。
- **LLM 兜底**：内网模型连不上时，用模板生成解释文本，Demo 照常跑（`MOCK_LLM=1` 开关）。

## 5. 多维比价规则引擎

| 维度 | 比价基准 | 异常判定（示例阈值，可调） | 对应赛题重点 |
|---|---|---|---|
| 集采落地价 | 集采中标价 | 实际采购价 / 中标价 > 1.5 | 集采落地 + 预警 |
| 历史价 | 该品种历史均价/中位数 | 环比涨幅 > 30% | 动态监测 |
| 跨区域 | 同通用名同厂家周边省份均价 | 偏离 > 50% | 动态监测 + 地区价差 |
| 同类替代 | 同类可替代药价格带 | 显著高于替代药 + 替代药挤占中选药 | 集采落地 |
| 采购量落地 | 约定采购量 | 完成率 < 60% | 集采落地跟踪 |
| 标准化冲突 | 统一医保编码 | 同"四同药品"不同名/不同价 | 数据标化治理 |

**风险分级**：综合各维度偏离度加权 → 评分 → **高 / 中 / 低**三级（高=红、中=橙、低=黄）。
**最终定级 = 规则风险分 × 对抗校验置信度**：规则先给初判，§3.2 对抗校验再用置信度（0–100%）修正——高置信才升级预警、低置信自动消解为误报，避免"只看阈值"的误判。

## 6. Mock 数据与植入异常（真实感优先，体现稳定性）

### 6.1 数据真实感原则（让评委信服"接真实数据即可上线"）

- **真实品种**：用真实存在的通用名/商品名/厂家/规格/剂型组合，覆盖药品 + 高值耗材。例：
  - 药品：阿莫西林胶囊、注射用头孢曲松钠、阿托伐他汀钙片、硫酸氢氯吡格雷片、二甲双胍缓释片、奥美拉唑肠溶胶囊、孟鲁司特钠片……
  - 耗材：冠脉药物洗脱支架、人工髋关节、一次性使用留置针……
- **编码格式真实**：医保药品编码按 **15 位**结构生成（首位类别 + 分类 + 流水校验位），耗材按"分类与代码"格式；保证"同四同药品同前段、不同厂家后段不同"的可比逻辑。
- **价格量级合理**：贴近现实（如普通口服药数元至数十元/盒，注射剂数十元，集采后大幅降价；支架经集采从上万元降至数百元）。
- **时间序列连续**：每个品种有 12–24 个月历史价，平稳段 + 在植入点出现异常拐点，趋势图自然。
- **区域真实**：用真实省/市名（如本省 + 周边 3 省），构造合理地区价差。
- **规模**：约 80–120 条主记录 + 配套历史序列，足够让大屏"有料"又可控。

> 不声称是官方真实数据，但**结构、量级、口径都贴近真实**，体现项目稳定性与可迁移性。

### 6.2 植入剧本（5 真异常 + 1 诱饵，覆盖赛题重点 & 演示对抗校验）

| # | 植入剧本 | 触发维度 | 对抗校验预期 | 映射赛题重点 |
|---|---|---|---|---|
| 1 | 某中选药在某医院采购价 **高于集采中标价 320%** | 集采落地价 | 反驳失败→**高置信确认** | 集采落地 + 异常预警 |
| 2 | 同通用名同厂家，本省较周边 3 省均价 **高 2.1 倍** | 跨区域 | 排除规格差异后→确认 | 动态监测 + 地区价差 |
| 3 | 某药历史平稳后 **环比突然跳涨 60%** | 历史价 | 排除政策调价后→确认 | 动态监测 |
| 4 | 集采中选药采购量 **仅达约定量 30%**，被高价替代药挤占 | 采购量 + 替代 | 确认（挤占证据充分） | 集采价格落地跟踪 |
| 5 | 同一药品 **不同商品名/规格未标化**、价差悬殊 | 标准化冲突 | `standardize_code` 归并后比价→确认 | 数据标化治理（内嵌工具体现）|
| 6 | **诱饵：看似异常实则合理**——某药价"偏高"，但因 **剂型/规格不同（缓释 vs 普通）不可比** 或 **处于集采切换过渡期** | 集采落地价/历史价 | **被对抗校验驳回→自动消解为误报** | 体现严谨、降误判（演示亮点）|

> 第 6 条是对抗校验环节的"压舱石"：Demo 演示智能体**主动驳回一个假阳性**，比只会报警更打动评委。

## 8. Demo 脚本（3–5 分钟）

1. **痛点开场**：药价虚高、地区价差、集采不落地、数据孤岛。
2. **监测大屏**：价格数据流入，系统自动标红异常，KPI 卡片一览。
3. **点开异常**（"采购价高于中标价 320%"）→ 展示**智能体推理轨迹**：自主调用比价工具取证 → LLM 生成「风险等级：高；依据：同通用名同厂家周边 3 省均价×0.3；疑似未按中标价采购」。
4. **对抗校验亮点**：智能体启动红队自我反驳（排除规格差异/政策调价）→ 置信度升至 **92%** → 升级高置信预警；随后点开**诱饵案例**，演示智能体**主动驳回一个假阳性**（剂型不同不可比）→ 自动消解、记录理由。
5. **一键生成核查工单** → 推入闭环看板。
6. **模拟处置** → 价格修正 → 智能体**复核** → **闭环关闭**（SQLite 留痕、刷新不丢）。
7. **收尾**：覆盖赛题 4 重点方向 + 真·Agent 闭环 + 对抗校验降误判 + 接真实数据即可上线。

## 10. 风险与降风险

| 风险 | 措施 |
|---|---|
| 内网 LLM 连不上 | `MOCK_LLM` 开关，模板生成解释，Demo 不依赖在线模型 |
| 检测翻车 | 检测全确定性规则 + 预置植入异常，必中 |
| 误报打扰监管 | **对抗校验环节**算置信度，自动消解假阳性 |
| 时间不够 | 严格 YAGNI；先打通"一条异常的完整闭环"，再加其余剧本 |
| 现场演示故障 | 提前录屏备份；数据预置 + SQLite 持久化可复现 |
| 术语不专业 | 业务同学校核话术；附术语表（见 §12） |
